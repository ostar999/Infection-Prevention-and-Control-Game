# 🏥 传染病防护知识培训游戏

基于《常见传染病和感染性疾病的防护要求》开发的交互式培训工具，面向医护人员及感控相关人员，通过游戏化方式帮助掌握 24 种常见传染病的防护装备选择、传播途径分类和隔离方式判断。

## ✨ 功能特性

### 🛡️ 防护装备选择（模式一）

根据给定疾病及其传播途径、隔离方式，从 9 种防护装备中选出正确的基本防护 + 可选防护组合。

- 正确选中所有基本装备且无多余选择：+60 分，可选装备额外加分
- 正确选中基本装备但有多余选择：+30 分，多余项扣分
- 遗漏基本装备或选错：扣分
- 每轮 5 题，随机抽取疾病

### 🦠 传播途径分类（模式二）

根据疾病名称和传播途径描述，在 6 种隔离分类中选择正确的隔离类型。

- 6 种分类：接触隔离、飞沫隔离、空气隔离、接触+飞沫隔离、接触+飞沫+空气隔离、特殊隔离
- 限时 8 秒作答，正确 +20 分，错误 -10 分
- 每轮 10 题，随机抽取疾病

### ⚡ 快问快答（模式三）

涵盖单选题、判断题、多选题三种题型，综合考察防护知识。

- 单选题：传播途径、隔离方式
- 判断题：防护装备是否正确
- 多选题：基本防护要求包括哪些
- 限时 10 秒作答，连击加分机制
- 每轮 10 题，题库随机生成

### 📊 学习统计

- 累计总分、累计答题数、正确率
- 最高连击记录
- **错题本**：自动记录错题，包含问题、你的回答、正确答案、时间和模式
- 数据持久化存储在浏览器 localStorage

### 📖 知识速查表

- 24 种传染病完整对照表
- 包含：疾病名称、传播途径、隔离方式、基本防护要求、可选防护用品
- 支持关键词搜索

## 🎮 游戏机制

| 机制 | 说明 |
|------|------|
| **连击系统** | 连续答对累计连击数，模式三中连击越高加分越多（最高 +25） |
| **倒计时** | 模式二 8 秒、模式三 10 秒，超时自动判错 |
| **视觉反馈** | 绿色=正确、红色=错误、黄色=遗漏、动画震动/弹出效果 |
| **评级系统** | 根据正确率给出 🏆优秀 / 👍良好 / 📚加油 / 💪需努力 |

## 🚀 使用方法

### 直接打开

1. 用浏览器打开 `index.html` 即可使用
2. 无需安装任何依赖，无需网络连接
3. 所有数据存储在浏览器本地

### Python 环境（可选）

仅当需要处理 backups 目录中的 xlsx/docx 参考文件时使用：

```bash
python -m venv .venv
source .venv/bin/activate  # macOS/Linux
pip install openpyxl python-docx lxml
```

## 📁 项目结构

```
.
├── index.html              # 主应用（单文件，包含 HTML/CSS/JS）
├── references/
│   └── 常见传染病和感染性疾病的防护要求.md   # 参考知识文档
└── .gitignore               # 告诉 Git 忽略哪些文件（见下方说明）
```

> **`.gitignore` 是什么？** 它告诉 Git **不要追踪**某些文件/文件夹，避免把无关内容提交到仓库。本项目忽略的内容包括：
>
> | 忽略项 | 原因 |
> |--------|------|
> | `.venv/` | Python 虚拟环境，体积大且因人而异，每人应自己创建 |
> | `backups/` | 参考数据的备份文件，非项目运行必需 |
> | `.DS_Store`、`Thumbs.db` 等 | 操作系统自动生成的无关文件 |
> | `node_modules/`、`__pycache__/` | 包缓存和编译缓存，可随时重新生成 |
> | `.claude/` | Claude Code 的本地配置，属于个人工具配置 |

## 🛠️ 技术栈

- **前端**：原生 HTML/CSS/JS，无框架依赖
- **数据持久化**：localStorage
- **样式**：CSS Grid + Flexbox，响应式布局
- **Python 工具**（可选）：openpyxl、python-docx、lxml
